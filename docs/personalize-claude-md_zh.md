# 配置个人 CLAUDE.md

> [English](personalize-claude-md.md)

Muse 是一个整体助理——同时理解用户的身体状况、日常事务、财务状况、关心的人与生活全貌。它不会按话题切换角色，而是始终基于完整背景信息作出回应。

为实现这一点，Muse 启动时会读取归档根目录下的 `CLAUDE.md`。该文件是用户向 Muse 提供的**自传式简介**。

**重要说明：** 模板对所有人生阶段保持中性——学生、在职人员、自由职业者、全职父母、退休人士、创业者均适用。不适用的章节直接删除，无需强行填写。

---

## 如何生成此文件

### 对话引导（推荐）

点击顶部导航栏的「**整理档案**」按钮（curator 模式）。Muse 会开启一个会话，先梳理 archive 现状，再带你过 `CLAUDE.md` 里空着的章节——逐题提问、通过 `Edit` 把答案写回文件。任意章节可回复「跳过」或「先不填」。涉及隐私的章节（财务 / 健康）优先询问量级或定性描述，精确数字由用户自愿补充。

如果 `CLAUDE.md` 还不存在，顶部导航栏会出现「未配档案」橙色提示，点击同样会跳到上述流程。

### 使用安装脚本（备选，交互较少）

`scripts/install-{linux,macos}` 首次运行时将询问是否：

1. 在 archive 下建好 6 个子目录（`health/` `work/` `money/` `people/`
   `notes/` `archives/`），每个带 README 说明
2. 问 7 个开放式信息采集问题：
   - 怎么称呼你
   - 出生年份（或年龄段）
   - 现住地
   - 一周大部分时间在做什么（学习 / 工作 / 自由职业 / 照护 / 退休 ……）
   - 一句话当前人生阶段
   - 今年主目标
   - 当前最关心的健康问题
3. 把答案直接 写入 CLAUDE.md 对应字段
4. 提示你接下来该把哪些原件放入哪个目录

跳过初始问卷也可以，所有问题按 Enter 跳过即可。后续可随时通过顶部导航栏的「整理档案」按钮回来继续填写。

### 手动创建

```bash
cp scripts/templates/default-CLAUDE.md ~/muselab-archive/CLAUDE.md
sed -i "s/%DATE%/$(date +%Y-%m-%d)/" ~/muselab-archive/CLAUDE.md
# 子目录骨架——每个子目录同时带 README.md（中文）和 README.en.md（英文），
# 这里只保留中文 README，避免把英文 README 混入归档
for sub in scripts/templates/archive-skeleton/*/; do
  name=$(basename "$sub")
  mkdir -p ~/muselab-archive/"$name"
  cp "$sub/README.md" ~/muselab-archive/"$name"/README.md
done
```

## 模板的 6 个章节

CLAUDE.md 模板设计是极简骨架 —— 每次对话都会加载进 Muse 的上下文，所以不放任何引导文字或选项列表，只保留字段。

| 章节 | 填写内容 |
|---------|--------|
| **1. 我是谁** | 称呼 / 出生年份 / 居住地 / 常用语言 / 同住情况 |
| **2. 我现在主要在做** | 一句话当前人生阶段 / 主要在做 / 做了多久 / 这一年最想做成 / 可能要做的大决定 |
| **3. 钱** | 收入来源 / 资产负债量级 / 当前关注 / 风险偏好 |
| **4. 身体** | 大致情况 / 上次体检 / 在用药 / 训练 / 关注问题 / 睡眠 |
| **5. 我关心的人** | 关键关系 / 现在需要关注的人 / 近期重要事 |
| **6. 当前所思所想** | 最大的烦恼 / 在做的事 / 想做但没动手的 |

子目录归档（`health/` / `work/` / `money/` / `people/` / `notes/` / `archives/`）由 Muse 通过 Read 工具按需访问，CLAUDE.md 不需要列出索引。每个子目录的 `README.md` 描述了放什么、Muse 在该领域的约束（如健康不下诊断、投资不预测涨跌等）。

## 子目录结构（6 个，通用于所有人群）

| 目录 | 适合存放 | 学生 | 在职 | 自由职业 | 全职父母 | 退休 |
|------|--------|------|------|---------|---------|------|
| `health/` | 身体相关 | 入学体检 | 年度体检 | 同左 | 自己及子女 | 慢病管理 |
| `work/` | 当前从事的事务 | 论文/申请材料 | 简历/项目 | 作品集/客户 | 育儿记录 | 当前活动 |
| `money/` | 财务相关 | 月度预算 | 收支/储蓄 | 报税/应急资金 | 家庭预算 | 现金流 |
| `people/` | 关心的人 | 父母/朋友 | 伴侣/同事/父母 | 同左 | 配偶/子女/双方父母 | 伴侣/子女/老友 |
| `notes/` | 杂项笔记 | 通用 | 通用 | 通用 | 通用 | 通用 |
| `archives/` | 原件归档 | 通用 | 通用 | 通用 | 通用 | 通用 |

每个目录均附有各自的 `README.md`，包含针对不同人生阶段的具体建议。

## 关键设计原则

### Muse 是一个整体助理，而非多个角色

跨领域决策是 Muse 的核心价值所在。示例：**父母刚完成心脏支架手术 + 用户今年的现金流状况 + 未来几年是否计划换工作 → 是否应为父母升级香港医疗保险？** 多角色模型会将此拆分给三个独立专家，产生三套互不连贯的建议；统一助理才能给出真正契合用户实际情况的综合方案。

### 模板保持中立

所有措辞、目录名、引导问题都避免预设任何特定人生阶段：

- `work/` 而非 `career/`（学生和退休人士同样适用）
- `money/` 而非 `investment/`（涵盖预算、助学贷款、养老金、FIRE）
- `people/` 而非 `family/`（也适合单身 / 未婚 / 只有朋友圈的人）
- 「你一周里大部分时间在做什么」而非「你的工作是什么」

### 行为约定不在 CLAUDE.md 里

跨领域的行为约束（涉及健康必须引用指南、不做诊断；涉及钱不预测涨跌等）放在每个子目录的 `README.md` 里（如 `health/README.md` / `money/README.md`）。Muse 进入对应目录时会读到。这样 CLAUDE.md 本身保持极简骨架，只放你的事实档案。

## 维护节奏

| 触发时机 | 更新位置 |
|------|--------|
| 体检后 | §4，PDF 放入 `health/` |
| 学业 / 工作 / 业务变化 | §2 |
| 重大财务变化 | §3，记录放入 `money/` |
| 关心的人发生重大变化 | §5 |
| 定期回顾 | 每半年回顾一次，删除不再适用的内容 |

## 隐私与安全

- 强烈建议为 muselab 归档目录启用文件系统加密（macOS FileVault / Linux LUKS）。
- 不要将归档目录同步至 OneDrive / Google Drive / Dropbox 等第三方云存储。
- 涉及他人的信息可进行脱敏处理（以「父亲」/「M」代替真实姓名）。
- 密码、身份证号、银行账号请存入专用密码管理器，不要写入归档目录。
- 远程备份请使用 [restic](https://restic.net) 或 [borg](https://borgbackup.org)，并启用端到端加密。
